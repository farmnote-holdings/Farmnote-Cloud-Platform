# AI駆動型デイリーオペレーション設計書

## 1. システムの意義
日々のPDCAサイクルを確実に回すことで、やるべきことを確実に行える組織づくりと、生産性の向上を実現する。

## 2. 前提となる戦略
本ワークフローは、以下の戦略文書で定義された構想を、牧場の現場レベルで具現化するものである。
- **[グループ長期経営戦略〜Farmnote Data Agent戦略〜.md](../.../../../executive/20_Strategy/グループ長期経営戦略〜Farmnote%20Data%20Agent戦略〜.md)**
- **[Farmnote Cloud Platform Ver.3 製品戦略.md](../.../../../executive/20_Strategy/Farmnote%20Cloud%20Platform%20Ver.3%20製品戦略.md)**

特に、`Farmnote Data Agent`および`Farmnote Assistant`を中核に据え、AIが担当者と管理者を有機的に繋ぐことで、データ入力の負荷を最小化し、データに基づいた意思決定を最大化することを目指す。

## 3. 登場人物と役割
- **担当者**: 日々の作業実行者。スマートフォンアプリとAIアシスタント（特に音声）を活用し、作業記録の負担を軽減しながら、現場の状況をリアルタイムに共有する。
- **管理者**: 牧場全体の管理者。AIが提示するデータやサマリーに基づき、スタッフへの作業割り振り、日々の進捗確認、そして翌日の計画策定といった意思決定を行う。

## 4. ワークフロー：AIを活用した日次PDCAサイクル

### Plan（計画）：管理者による翌日作業計画
管理者は毎日、AIの支援を受けながら翌日の作業計画を策定・承認する。

1.  **AIによる計画素案の自動生成**:
    `Farmnote Assistant`は、以下の3つの情報源から翌日の作業予定リストの素案を自動で作成する。
    -   **① システムからの自動予定**: 妊娠鑑定後の分娩予定、分娩後の発情予定など、個体イベントに基づきシステムが自動生成するタスク。
    -   **② プロトコルに基づく自動予定**: ワクチンプログラムや受精卵移植プログラムなど、事前にユーザーが設定したプロトコルをトリガーとして自動生成されるタスク。
    -   **③ 管理者の指示に基づくルーティン・個別タスク**:
        -   搾乳機の洗浄や牛舎の掃除といった定型的なルーティンワーク。
        -   個別の牛を指定した治療指示など、管理者が都度指示するタスク。

2.  **AIによるインテリジェントな作業割り振り**:
    -   管理者が「Aさんを明日の搾乳牛舎担当にする」と指示すると、AIは事前に登録された牧場のコンテキスト（RAG）を参照する。
    -   「搾乳牛舎で行うべきルーティンワーク」の定義と、「搾乳牛舎に現在いる牛のリスト」を組み合わせ、Aさんのタスクリストを自動で生成する。

3.  **管理者による確認・修正・承認**:
    -   AIが生成した担当者別の作業予定リストを管理者が確認し、必要に応じて修正・追加を行う。
    -   管理者がリストを承認すると、翌日の担当者の作業予定が確定する。

### Do & Check（実行と記録）：担当者による作業と音声メモ
担当者は自身のスマートフォンやスマートウォッチで今日の作業を確認し、実行する。作業記録は、スマートフォンからの直接入力か、AIの活用により現場作業を中断することなく行える。

1.  **作業記録の方法**:
    -   **① スマートフォンアプリ**: 従来のアプリと同様に、割り当てられた予定に対して完了フラグを立てることで作業を記録する。
    -   **② 音声入力（ハンズフリー）**: スマートフォンやスマートウォッチに話しかけることで、作業内容を「音声メモ」として一時的に記録する。
    -   **③ 画像・動画メモ**: スマートフォンで撮影した写真や短い動画をメモとして記録する。音声メモと組み合わせることで、より正確な情報を伝達する。

2.  **音声メモ機能**:
    -   **目的**: 現場担当者が手袋を外したり、スマートフォンを操作したりすることなく、発見したことや完了した作業を気軽に記録するための一次保存場所。
    -   **プロセス**:
        -   担当者が「1000番、発情見つけた」と音声入力。
        -   AIはその音声をテキスト化し、「音声メモ」にタイムスタンプと共に記録する。
        -   同時に、バックグラウンドでAIが「1000番」の個体データを参照し、「発情」というイベントと紐づけ、正式な活動記録として登録できる形に整形・提案する。
        -   「牛舎の清掃完了」といった個体に関連しない作業や、「抗生物質が足りない」といった発注依頼なども同様にメモとして記録できる。

3.  **画像・動画メモ機能**:
    -   **目的**: 跛行の様子や設備の故障箇所など、音声やテキストだけでは伝えきれない視覚的な情報を補完する。
    -   **プロセス**:
        -   担当者が個体の写真を撮影すると、AIが画像から個体を特定（例：耳標を読み取り）し、「個体XXXに関するメモ」として自動で紐づける。
        -   動画を撮影し、「1234番、右後ろ脚を跛行」のように音声メモと組み合わせることで、より状況が伝わりやすい記録を作成できる。

### Action（評価・改善）：管理者による日次レビューと翌日への反映
一日の終わりに、管理者はAIがまとめたレポートを通じて、牧場全体の状況をレビューし、翌日の計画へと繋げる。

1.  **担当者による作業記録の確認（日報）**:
    -   担当者は、一日の終わりに自身が記録した「音声メモ」の一覧と、「スマートフォンアプリで直接記録した作業」の両方を確認し、内容の過不足を修正する。

2.  **管理者による日次レビュー**:
    -   AIが全担当者の「音声メモ」と、システム上の「作業計画の進捗」を統合した日時レポートを自動生成する。
    -   管理者はレポートで以下の内容を確認する。
        -   **計画の達成状況**: 誰がどの作業を完了し、何がやり残されているか。
        -   **現場からの報告**: 担当者のメモから、発情発見などの定型報告だけでなく、「〇〇に困っている」「〇〇という発見があった」といった定性的な情報も把握する。
        -   **主要KPIの進捗確認**: その日の搾乳量や個体販売額といった主要KPIの実績を、事前に設定した計画データと比較し、進捗状況を確認する。
        -   **AIからのアドバイス**: AIが計画と実績のギャップを分析し、「搾乳量が計画を下回っています。原因として考えられる〇〇を確認してください」といった具体的なアドバイスを提示する。
    -   管理者は問題のないメモにチェックを入れることで、正式な活動記録として確定・保存する。（音声メモはあくまで一時保存場所）

3.  **翌日計画への反映と経営状況の確認**:
    -   翌日の予定を作成する際、AIは「今日のやり残しタスク」を提示し、翌日に持ち越すか、あるいはキャンセルするかを管理者に問いかける。
    -   レビューのタイミングで、その日の搾乳量や個体販売額などの実績データと、事前に設定した経営計画との比較進捗も提示される。
    -   これにより、「計画の作成」が「日々の振り返り」と一体化し、管理者は常に現状を把握しながら次のアクションを考えることができる。

## 5. ワークフローを支える主要機能

### 牧場のコンテキストを理解するRAG (Retrieval-Augmented Generation)
- 牧場の牛舎構造、担当者のスキルや役割、経営方針、独自の作業プロトコルなどを事前にテキストデータとして登録し、RAGの知識ベースとして保存する。
- `Farmnote Assistant`は常にこのRAGを参照し、その牧場のコンテキストに合わせた最適な作業指示やアドバイスを生成する。
- **コンテキスト登録のサポート機能**: 導入時の負荷を軽減するため、AIが対話形式で「この牛舎の主な用途は何ですか？」「ワクチンのプロトコルについて教えてください」といった質問を投げかけ、管理者が答えるだけでコンテキストデータが自動生成されるオンボーディング機能を提供する。

### AIエージェントによる有機的な連携
- 本ワークフローは、担当者（現場）、音声・画像メモ（非構造データ）、管理者、そして`Farmnote`（構造化データ）という要素を、AIが有機的に連携させることで成立する。
- AIが構造化データとテキスト・画像データを融合して解釈・処理することで、単なるデータ入力の効率化に留まらない、牧場のPDCAサイクルそのものを進化させる。

## 6. ワークフロー強化機能

### リアルタイム異常検知とタスク自動割り振り
一日の終わりのレビューだけでなく、リアルタイムでの問題発見と対応を迅速化する。
- **機能**: センサーデータ（Farmnote Color等）や作業計画と実績をAIが常時監視し、異常を検知した場合に即座に関係者へアラートを送信する。（例：「個体XXXの活動量が低下しています」「牛舎Aの清掃が計画時刻を過ぎても完了報告されていません」）
- **タスク自動割り振り**: アラート発生時、事前に設定したルールに基づき、AIが自動で担当者に確認・対応タスクを割り振る。割り振りルールは「担当者のみ」「担当者と管理者」「勤務者全員」など、状況に応じて柔軟に設定可能とする。

### ゲーミフィケーションによる継続利用の促進
担当者が楽しみながらデータ入力を継続できる仕組みを導入し、エンゲージメントを高める。
- **機能**: 作業記録や重要な発見（疾病の早期発見など）といったアクションに対して「アクティブ度」としてスコアを付与する。
- **動機付け**: スコアが一定に達するとレベルアップし、画面に祝福の画像やメッセージを表示するなど、ポジティブなフィードバックを提供する。チーム内でのランキング表示なども可能とする。

### 消耗品の在庫管理と自動発注提案
現場の気づきを、発注業務までシームレスに繋げる。
- **機能**: 「抗生物質が足りない」といった音声メモをAIが検知し、`Farmnote Inventory`（在庫・購買管理アプリ）と連携する。
- **半自動化**: AIが在庫状況を確認し、購買担当者や管理者に「抗生物質Aを10本発注しますか？」といった具体的な内容でプッシュ通知を送信。承認するだけで発注が完了する。

## 7. 実現可能性の評価と課題

### 実現可能性
全体として、本ワークフローは現在のAI技術とアプリケーション開発レベルで**十分に実現可能**な構想である。

-   **技術的実現性**:
    -   音声認識、自然言語処理（NLP）、RAG、スマートフォン/スマートウォッチ連携といった中核技術は確立されている。
    -   ただし、「牧場の騒音環境下における音声認識の精度向上」や、「汚れた耳標など、不確定要素の多い画像からの安定した個体識別」は、継続的な改善が必要な挑戦的課題となる。
-   **運用的実現性**:
    -   「音声/画像メモ」という入力負荷の低い手法と、「管理者による最終承認」というプロセスを組み合わせることで、現場の負担を軽減しつつ、データの正確性を担保するフローは、現実的かつ効果的である。

### 克服すべき課題
本ワークフローを成功させるためには、以下の課題を克服する必要がある。

-   **導入・定着のハードル**:
    -   **課題**: 現場担当者のITリリテラシーの差や、新しいワークフローへの変化に対する心理的抵抗が、導入・定着の障壁となる可能性がある。
    -   **対策**: 誰でも直感的に使えるストレスのないUI/UX設計を追求するとともに、導入初期の丁寧なオンボーディングや、伴走型でのサポート体制が不可欠となる。
-   **コンテキストデータ（RAG）の整備・維持**:
    -   **課題**: 本システムの性能は、RAGに登録されるコンテキストデータの質と鮮度に大きく依存する。この「牧場の憲法」とも言えるデータを誰が、どのタイミングで、どのように更新していくのか、という運用が最も重要な成功要因であり、同時に最大のハードルとなりうる。
    -   **対策**: 対話形式での登録サポート機能で初期導入の負荷は軽減しつつも、継続的に情報をメンテナンスしていくための運用フローを確立する必要がある。
-   **AIの判断に対する信頼性の担保**:
    -   **課題**: AIがなぜそのタスクを提案したのか、なぜそのアラートを発したのかという判断根拠が不明瞭（ブラックボックス）だと、ユーザーはAIを信頼しなくなり、結果として使われなくなるリスクがある。
    -   **対策**: AIの判断根拠（例：「センサーデータXXの低下を検知したため」「プロトコルYYに基づき」）を可能な限り平易な言葉で提示し、判断の透明性を高める「説明可能性（Explainable AI）」の視点が重要となる。
